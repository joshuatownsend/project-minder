## MCP & Config Security

*Inspired by [mcpware/cross-code-organizer](https://github.com/mcpware/cross-code-organizer), which connects to live MCP servers and runs multi-layer threat analysis. MCP servers are a growing attack surface — malicious tool descriptions can exfiltrate credentials, inject prompts, or silently change behavior between scans ("rug-pulls").*

- [x] **MCP security scanner** — Connect to each configured MCP server (from `.mcp.json` + `settings.json`) and introspect tool definitions via JSON-RPC 2.0, then run a multi-layer threat scan: (1) **Deobfuscation layer** — 8 passes before pattern matching: strip zero-width Unicode, remove bidirectional text controls, strip HTML comments, Base64-decode encoded blocks, normalize escape sequences, detect leetspeak substitutions; (2) **Pattern layer** — 60+ regex rules across threat categories: prompt injection, tool poisoning, credential harvesting (flag parameter names like `webhook_url`, `ssh_key`, `api_token`), data exfiltration, code execution triggers, sensitive file access, persistence mechanisms; (3) **Hash layer** — SHA256-fingerprint each tool description at first scan; alert when a definition changes unexpectedly on re-scan ("rug-pull" detection). Safety guards: 15-second timeout per server, 500-tool cap, 50KB description limit, 5 concurrent connections, stdio + HTTP/SSE transport. Surface findings in the Config browser's MCP section as a "Security" tab: severity badges (critical / high / medium / low), deobfuscated preview, rug-pull diff, last-scanned timestamp. Hash baselines persisted to `.minder.json` or a SQLite table. New module: `src/lib/scanner/mcpSecurityScanner.ts`. Ref: `src/security-scanner.mjs` + 8-pass deobfuscation + 60+ patterns + SHA256 rug-pull in `mcpware/cross-code-organizer`.

- [x] **Task queue & cron scheduler** — A task queue for composing and dispatching work to Claude Code directly from the dashboard. A background dispatcher (Node.js child process spawned by the Next.js server, or a dedicated `src/lib/dispatcher.ts` loop running as a `globalThis` singleton) picks up `status='pending'` tasks, honors `requires_approval` → `awaiting_approval` promotion, and spawns `claude` CLI as a child process. Ref: `DECISION:` + `INBOX:` parsing (fence-aware, skips triple-backtick blocks) + stdin injection + partial UNIQUE index in `build-your-own-dashboard-prompt.md`.
- [x] **Emergency stop (kill-switch for dispatched agents)** — A red "Emergency Stop" button prominently placed on the dashboard that, when confirmed, SIGTERMs only the Claude Code child processes that were spawned by the Project Minder dispatcher — leaving any Claude Code sessions the user started manually in their terminal unaffected.
