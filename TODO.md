## Memory Observatory

*Inspired by [Nicolas Bustamante's "Agent Memory Engineering"](https://x.com/nicbstme/status/2050301124314563025), comparing how Claude Code, Codex CLI, and Hermes implement agent memory. Key findings: clever architectures (vector DBs, knowledge graphs, dedicated memory agents) all lost to **LLM + typed markdown + bash**; the interesting axis is the read/write **discipline** the agent follows, not the data structure; memory is a **hint surface, not authority** — every body read should be wrapped in an age-aware "verify before recommending" reminder. Phase 1 (Freshness + Telemetry + Budget + MEMORY.md index awareness on `/memory`) is in active planning per the approved plan at `~/.claude/plans/i-recently-read-this-temporal-crane.md`; the entries below are the deferred follow-on phases, in priority order.*
